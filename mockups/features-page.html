Features page — 5 mockups

1 · Categorized cards 2 · Docs sidebar 3 · Compact list 4 · Bento + categories 5 · Segmented tabs
Option 1
Categorized cards
Closest to the docs sidebar, rebuilt as scannable cards. Each category is a labelled block with a 3-up grid of icon + title + one-line value. Calm, roomy, very easy to skim. Best when you want every feature visible at once.
Option 2
Docs sidebar + content
A literal nod to docs.linkrunner.io: a sticky category rail on the left with icon nav, feature cards on the right. Feels like product documentation, signals depth and seriousness. Best on desktop; the rail collapses on mobile.
Option 3
Compact list rows
Dense and efficient. Each category is a bordered list; every feature is a single row (icon, title, description, chevron). Mirrors the sidebar's vertical rhythm most closely and fits a lot without scrolling much. Best when the list is long.
Option 4
Bento highlights + categories
Keeps a hint of the current bento: three marquee tiles up top (one filled in brand purple) for the headline capabilities, then tidy categorized mini-grids below. Best when you want to lead with a few hero features before the full list.
Option 5
Segmented tabs
One category visible at a time via a segmented switcher (Core Features / SDK-less / API Reference). Keeps the page short and focused, and the click is a tiny bit of engagement. Try the buttons below. Best when you want a clean, controlled first impression.
